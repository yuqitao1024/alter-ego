# Example Feature Development Workflow

1. Confirm the request scope, repository context, and branch target.
2. Inspect the current implementation and identify the minimal change set.
3. Write or update tests before changing production code.
4. Implement the change and keep the branch focused on the requested behavior.
5. Run the relevant verification commands and capture the result.
6. Summarize the change, risks, and any follow-up work for the operator.
